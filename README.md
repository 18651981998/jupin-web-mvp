# 聚品服饰 · 数字化管理平台（网页版 MVP）

服装企业「样版全生命周期管理」系统的**部门门户网页版原型**，由原 VBA 系统（WPS/Excel + Access）拆解重构而来。

> 本项目是 **MVP 原型**：用本地生成的模拟数据还原原系统的核心交互与部门职能划分，验证"网页版长什么样、怎么组织"。生产环境需接入真实数据库与登录权限（见文末方案）。

**作者**：A0_0CN涛声依旧 ｜ **版本**：V26.0408（每次修改版本号尾数 +1，如 V26.0407 → V26.0408）

## 🚀 本地运行

直接用浏览器打开 `index.html` 即可；或起一个本地静态服务器（推荐，避免个别浏览器对本地文件的限制）：

```bash
python -m http.server 8080
# 然后访问 http://localhost:8080

# 或使用 Node
npx serve .
```

## 🔌 接入真实数据库（采购部已就绪）

原 VBA 系统后台是 `X:\A返款登记\数据存储.accdb`（Access + ADODB）。网页端无法直接连 Access，需经后端桥接。`server/` 已提供完整方案：

**方案 A：本地 Access 桥接（保留原库，改动最小）** —— 推荐先用这个跑通
- `cd server && npm install && npm start`（默认 JSON 回退模式，立刻可用）
- 前端 `index.html` 顶部加 `window.JP_API_BASE = 'http://<后端地址>:8787'`，采购部即自动切到真实数据
- 要连真实库：设置 `DB_PATH="X:/A返款登记/数据存储.accdb"` 启动，按 `数据存储.accdb` 实际**表名/字段名**核对 `index.js` 中 `TABLES` 与 SQL（VBA 中确认的表：`拍摄数据`；供应商/入库表名请对照知识库核对）

**方案 B：迁移到云数据库（Supabase / Postgres，免费层够用）**
- 把 `.accdb` 表结构迁移为 Postgres 表（样版表、客户表、摄影师表、供应商表、入库表、部门/权限表）
- `data.js` / 各功能模块改为调用 Supabase REST API 拉取数据
- 用 Supabase Auth 实现原系统的部门与功能权限
- 用 Supabase Storage 存放样版图（替代原系统的图片交付/退版流程）

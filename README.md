# 聚品服饰 · 数字化管理平台（网页版 MVP）

服装企业「样版全生命周期管理」系统的**部门门户网页版原型**，由原 VBA 系统（WPS/Excel + Access）拆解重构而来。

> 本项目是 **MVP 原型**：用本地生成的模拟数据还原原系统的核心交互与部门职能划分，验证"网页版长什么样、怎么组织"。生产环境需接入真实数据库与登录权限（见文末方案）。

---

## 🏢 门户结构（按行业标准划分）

| 板块 | 二级业务域 | 三级功能页（节选） |
|------|-----------|-------------------|
| 🏠 首页 | 数据概览 + 部门导航 + 待办 | 仪表盘 |
| 📷 摄影部 | 样版管理 / 样版生命周期 / 统计报表 | 样版查询·交付 ✅、样版分配、拍摄进度、图片交付、入库登记、拍完退版、拍摄量统计、摄影师绩效 |
| 💰 财务部 | 费用结算 / 对账管理 / 财务报表 | 拍摄费用结算、返款登记、客户结账、欠款对账、收款管理、结算明细、利润核算 |
| 👥 人事部 | 员工管理 / 薪资管理 / 考勤管理 | 员工档案、摄影师管理、薪资核算、绩效奖金、考勤记录、请假审批 |
| 📦 仓库 | 库存管理 / 流转管理 / 盘点管理 | 样版入库、样版退版、库存查询、分配出库、归还入库、定期盘点、盘盈盘亏 |
| 📊 运营部 | 客户管理 / 订单管理 / 数据运营 | 客户档案、客户订单、订单录入、订单跟踪、业务看板、经营分析 |

- 共 **5 个部门 + 首页**，**34 个三级功能页**。
- 标记 ✅ 的为已上线的真实功能（摄影部·样版查询/交付，对应原 VBA `UserForm9`）；其余为「功能规划中」占位页，含行业标准说明，后续逐个实现。

## 📁 文件结构

```
jupin-web-mvp/
├── index.html   # 门户外壳（侧边栏 + 顶栏 + 主内容区）
├── styles.css   # 浅色专业主题样式（门户布局 + 组件）
├── menu.js      # 部门 / 二级 / 三级菜单树（行业标准）
├── data.js      # 模拟数据层（生成器，可替换为真实 API）
├── app.js       # 摄影部「样版查询/交付」功能模块（可挂载组件）
├── portal.js    # 门户框架：侧边栏渲染、hash 路由、首页/部门页/功能页
└── README.md
```

零外部依赖、零构建步骤，纯原生 HTML/CSS/JS（SPA + hash 路由），浏览器直接打开即可运行。

## 🚀 本地运行

直接用浏览器打开 `index.html` 即可；或起一个本地静态服务器（推荐，避免个别浏览器对本地文件的限制）：

```bash
python -m http.server 8080
# 然后访问 http://localhost:8080

# 或使用 Node
npx serve .
```

## 🔌 后续接入真实数据层（替换 Access）

原 VBA 系统后台是 `X:\A返款登记\数据存储.accdb`（Access + ADODB）。网页版建议改为云数据库，最小改动替换 `data.js` 中的数据获取逻辑：

**推荐方案：Supabase（Postgres，免费层够用）**
- 把 `.accdb` 表结构迁移为 Postgres 表（样版表、客户表、摄影师表、部门/权限表）
- `data.js` / 各功能模块改为调用 Supabase REST API 拉取数据
- 用 Supabase Auth 实现原系统的部门与功能权限
- 用 Supabase Storage 存放样版图（替代原系统的图片交付/退版流程）
